## 背景分析

### 目标用户画像

- **主要用户**：网文作者、小说创作者
- **技术背景**：非程序员居多，对 JSON/Markdown 不熟悉
- **核心需求**：专注写作、字数统计、敏感词检测
- **痛点**：VSCode 界面复杂、命令面板不直观

### 核心原则

1. **开箱即用** - 默认配置适合 90% 用户
2. **渐进式学习** - 先简单后复杂，不强迫学习
3. **容错设计** - 操作可撤销，错误可恢复
4. **可视化优先** - 能点击不输入，能选择不填写

## 当前使用门槛

### 高门槛（阻碍使用）

| 门槛 | 问题描述 | 影响程度 |
|------|---------|---------|
| VSCode 本身 | 代码编辑器界面，非写作工具 | ⭐⭐⭐⭐⭐ |
| 命令面板 | Cmd+Shift+P 不直观 | ⭐⭐⭐⭐ |
| Markdown 语法 | 需要学习 `#`、`---` 等 | ⭐⭐⭐ |
| Front Matter | YAML 格式陌生 | ⭐⭐⭐ |
| 配置文件 | JSON 格式编辑 | ⭐⭐⭐ |
| Git 概念 | 版本控制需额外学习 | ⭐⭐ |

### 中门槛（影响体验）

| 门槛 | 问题描述 |
|------|---------|
| 无新手引导 | 打开插件不知道从何开始 |
| 功能隐藏 | 很多功能不知道存在 |
| 无预览 | 不是所见即所得 |
| 快捷键 | 需要记忆 |

## 改进方案

### 第一阶段：降低入门门槛

> 优先级：P0（最高）
> 预期效果：新用户 5 分钟内开始写作

#### 1.1 新手欢迎页面

**触发条件**：首次安装插件打开工作区

**页面内容**：
```
┌─────────────────────────────────────────────┐
│  欢迎使用 Noveler 中文小说写作助手          │
├─────────────────────────────────────────────┤
│  [图片：插件功能预览]                        │
│                                             │
│  核心功能：                                  │
│  ✅ 实时字数统计（符合网文标准）             │
│  ✅ 敏感词自动检测                          │
│  ✅ 对话和人物高亮                          │
│  ✅ 智能格式化                              │
│                                             │
│  [🚀 创建示例项目]  [📂 打开现有项目]        │
│                                             │
│  □ 不再显示此页面                            │
└─────────────────────────────────────────────┘
```

**技术实现**：
- 使用 Webview 实现
- 存储用户偏好到 globalState
- 点击按钮触发相应命令

---

#### 1.2 初始化向导增强

**当前流程**：
```
输入小说名称 → 创建项目
```

**改进后流程**：
```
第 1 步：输入小说名称
        [我的武侠小说_______]

第 2 步：选择模板（可选）
        ○ 空白项目
        ● 包含示例内容（推荐新手）
        ○ 从模板创建...

第 3 步：基础设置
        引号样式：[「」 ▼]
        段落缩进：[✓] 启用
        敏感词检测：[✓] 启用

        [创建项目]
```

**示例项目内容**：
```
my-novel/
├── chapters/
│   └── 第一章 示例.md      ← 带注释说明
├── characters/
│   └── 示例人物.md         ← 演示人物模板
├── novel.jsonc
└── README.md
```

**示例章节内容**：
```markdown
---
title: "第一章 示例"
chapter: 1
wordCount: 0
status: "草稿"
---

# 第一章 示例

<!-- 这是一个示例章节，你可以删除这些内容开始写作 -->

清晨的阳光透过窗棂洒在青石板上。

「早安。」她微笑着说道。

<!-- 提示：
- 「」内的对话会自动高亮
- 状态栏会实时显示字数
- 保存时会自动格式化
-->
```

### 第二阶段：简化日常操作

#### 2.4 导出为纯文本

**用途**：方便复制到网文平台发布

**功能**：

#### 3.2 导出功能

**支持格式**：

| 格式 | 用途 | 复杂度 |
|------|------|--------|
| TXT | 网文平台发布 | 低 |

- 是否包含元数据

### 第四阶段：智能辅助

> 优先级：P3
> 预期效果：智能化写作体验

#### 4.1 写作目标提醒

**功能**：
- 设置每日/每周字数目标
- 状态栏显示进度
- 达成目标时庆祝动画

**配置**：
```jsonc
{
  "noveler": {
    "writingGoals": {
      "dailyTarget": 2000,
      "showProgress": true,
      "enableReminder": true
    }
  }
}
```

---

#### 4.2 常见问题自动检测

**检测项目**：
- 段落没有缩进
- 使用了英文标点
- 章节标题格式不规范
- 对话引号不匹配

**提示方式**：
- 温和的提示条
- 一键修复选项
- 可选择"不再提醒"

---

#### 4.3 AI 辅助写作（远期）

**可能的功能**：
- 智能续写建议
- 人物对话润色
- 情节建议
- 错字纠正

**实现方式**：
- 集成 LLM API
- 本地模型可选
- 用户可控开关

## 与传统工具对比

### Noveler vs WPS/Word

| 维度 | Noveler | WPS/Word |
|------|---------|----------|
| 字数统计 | ✅ 符合网文标准 | ❌ 含空格，不准确 |
| 敏感词检测 | ✅ 三级词库 | ❌ 无 |
| 语法高亮 | ✅ 对话/人物高亮 | ❌ 无 |
| 版本控制 | ✅ Git 友好 | ❌ 二进制文件 |
| 文件大小 | ✅ 纯文本，极小 | ❌ 二进制，较大 |
| 学习成本 | ⚠️ 需要适应 | ✅ 所见即所得 |
| 移动端 | ❌ 体验差 | ✅ 良好 |

### Noveler vs 专业网文工具

| 维度 | Noveler | 壹写/小黑屋 |
|------|---------|-------------|
| 版本控制 | ✅ 完整 Git 支持 | ❌ 无 |
| 扩展性 | ✅ VSCode 生态 | ❌ 封闭 |
| 云同步 | ⚠️ 需手动 Git | ✅ 内置 |
| 一键发布 | ❌ 无 | ✅ 支持 |
| 免费程度 | ✅ 完全免费 | ⚠️ 部分收费 |
